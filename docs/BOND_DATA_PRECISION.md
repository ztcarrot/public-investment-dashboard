# 债券数据精确获取方案

## 概述

已实现针对不同类型债券资产的精确数据获取策略，确保数据的准确性和可靠性。

## 数据获取策略

### 1. 可转债（11xxxx/12xxxx）- 精确K线数据 ✅

**接口**：`ak.bond_zh_hs_cov_daily(symbol)`

**数据特征**：
- ✅ 真实的历史交易价格（open, high, low, close）
- ✅ 完整的K线数据，包含成交量
- ✅ 每日更新
- ✅ 精确到小数点后3位

**代码识别**：
- `11xxxx`：上海市场可转债
- `12xxxx`：深圳市场可转债

**示例数据**：
```python
# Z华安转 (110067)
2026-03-06: close=106.999, volume=5750010
2026-03-05: close=107.014, volume=6998240
```

### 2. 债券型基金（如005350）- 精确净值 ✅

**接口**：东方财富基金API

**数据特征**：
- ✅ 精确的单位净值
- ✅ 完整的历史记录
- ✅ 每日更新
- ✅ 适用于所有债券型基金

**回退机制**：
- akshare 失败时自动使用东方财富基金API
- 这是债券型基金最可靠的数据源

### 3. 特别国债（如019789）- 专用处理 ✅

**方法**：
1. 从东方财富API获取当前价格
2. 基于年化收益率推算历史价格
3. 提供固定价格作为最后备份

**数据特征**：
- ✅ 准确的当前价格
- ⚠️ 历史价格为推算值（基于收益率）

### 4. 其他债券 - 收益率曲线推算 ⚠️

**接口**：`ak.bond_china_yield()`

**数据特征**：
- ⚠️ 基于国债收益率曲线推算价格
- ⚠️ 简化定价公式（仅作参考）
- ℹ️ 适用于趋势分析，不推荐用于精确计算

## 实现细节

### 代码结构

**文件**：`utils/data_fetcher.py`

**关键方法**：
```python
def get_bond_historical_from_akshare(code, start_date, end_date):
    """智能路由：根据债券类型选择数据源"""

    if code.startswith('11') or code.startswith('12'):
        # 可转债 → 精确K线数据
        return self._get_convertible_bond_data(code, start_date, end_date)
    else:
        # 其他债券 → 收益率曲线推算
        return self._get_bond_yield_curve_data(start_date, end_date)
```

### 数据流程

```
债券类型资产
    ↓
判断债券类型
    ↓
┌─────────────┬──────────────┬────────────────┐
可转债        债券型基金      特别国债        其他
(11/12开头)   (005350等)     (019789等)     (国债等)
↓             ↓              ↓              ↓
精确K线        东方财富API     专用API        收益率推算
(akshare)      (精确净值)      (当前价格)     (估算数据)
```

### 数据精度要求

- **高精度需求**：使用可转债（11/12开头）或债券型基金
- **趋势分析**：可使用收益率推算数据
- **精确计算**：优先选择有精确K线或净值的债券

## 已知限制

1. **可转债数据**：
   - 需要完整的6位代码
   - 部分退市债券可能无数据
   - 数据从2001年开始

2. **国债收益率**：
   - 推算价格为近似值
   - 不适用于精确计算
   - 仅供参考

3. **特别国债**：
   - 历史数据为推算值
   - 基于固定年化收益率
