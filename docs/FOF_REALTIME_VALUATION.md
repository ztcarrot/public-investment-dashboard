# FOF 基金数据获取优化说明

## 问题描述

016644（国泰瑞悦3个月持有债券FOF）等 FOF 基金的数据获取延迟 2-3 天。

## 问题原因

**FOF 基金**（Fund of Funds）是投资于其他基金的基金：
1. 需要等待底层基金净值公布（通常是 T+1）
2. 然后才能计算 FOF 自己的净值
3. 因此 FOF 净值公布通常是 T+2 或 T+3 天
4. **这是行业普遍现象，不是数据源问题**

## 解决方案

### 1. 新增数据源

在 `utils/data_fetcher.py` 中添加了多个备用数据源：

- **AkShare**：`get_fund_historical_from_akshare()`
- **天天基金**：`get_fund_historical_from_ttjj()`
- **实时估值**：`get_fund_realtime_estimate()` ⭐ 核心改进

### 2. 实时估值功能

添加了 `get_fund_realtime_estimate()` 方法，通过天天基金接口获取盘中实时估值：

```python
# 天天基金实时估值接口
url = f"http://fundgz.1234567.com.cn/js/{code}.js"
```

**实时估值 vs 确认净值对比**：

| 数据类型 | 更新时间 | 延迟 | 准确性 |
|---------|---------|-----|--------|
| 实时估值 | 盘中实时更新 | ~1 天 | 估算值 |
| 确认净值 | 收盘后公布 | 2-3 天 | 官方确认 |

### 3. 应用集成

在 `app.py` 的配置管理页面中，获取当前价格时优先使用实时估值：

```python
# 对于基金类型，优先使用实时估值
if code_type == '基金':
    realtime_data = fetcher.get_fund_realtime_estimate(code)
    if realtime_data and realtime_data.get('实时估值'):
        current_price = realtime_data['实时估值']
        st.caption(f"✨ 使用实时估值（更新于 {realtime_data.get('估算时间', '未知')}）")
    else:
        # 回退到历史数据
        ...
```

## 使用说明

### 对于 FOF 基金用户

1. **配置管理页面**：输入基金代码后，会自动显示实时估值
2. **数据看板页面**：历史数据仍使用确认净值（保证准确性）
3. **提示信息**：会显示使用的是"实时估值"还是"确认净值"

### 注意事项

- **实时估值不是确认净值**：两者会有一定偏差（通常 ±5% 以内）
- **盘中更新**：实时估值在交易日的 9:30-15:00 实时更新
- **周末/节假日**：非交易日实时估值不会更新

## 技术细节

### 实时估值接口

- **接口地址**：`http://fundgz.1234567.com.cn/js/{基金代码}.js`
- **返回格式**：JavaScript JSONP
- **数据字段**：
  - `gsz`: 实时估值
  - `gszzl`: 估算涨跌幅
  - `gztime`: 估算时间

### 多数据源优先级

基金类型数据获取优先级：
1. **AkShare**（可能更新更快）
2. **天天基金历史数据**（备用）
3. **东方财富**（默认，最稳定）

当前价格获取优先级：
1. **实时估值**（最快，~1天延迟）
2. **历史数据最新值**（备用，2-3天延迟）
